# 工作流程

把使用者提供的內容改寫成單人獨白腳本，輸出可直接餵給 `audio` skill 生成 mp3 的 `.txt` 檔。

## 輸出格式

固定使用 `[說書人]` 標記，一段一行，段落間空行：

```
[說書人] 今天要說的是一件關於 AI 如何改變軟體開發的事。

[說書人] 這件事大概從兩三年前開始發酵，但真正讓人感受到的，是最近這段時間。

[說書人] 先來說說背景。那時候大家還在討論 AI 能不能寫程式，現在已經在討論要怎麼跟 AI 一起工作了。
```

## 寫作風格

有聲書旁白 — 自然、平實，像一個人在為你娓娓道來。

- **自然口語**：像真人在說話，不是在朗讀文章
- **語調平穩**：不誇張、不煽情，保持一致的說書節奏
- **不要戲劇化腔調**：避免「話說」「且聽下回」「列位看官」「欲知後事如何」等說書人套語
- **不要把自己當第二個說話者**：不要用「各位聽眾」「你們覺得呢」等對話語氣，這是獨白，不是互動

## 改寫規則

1. **去掉所有 emoji 和表情符號** — TTS 會把 `:fire:` 念成「fire」非常突兀
2. **去掉 URL、`@mention`、Hashtag** — 有聲書用不到，念出來干擾
3. **長句拆短** — Edge TTS 中文長句容易語速忽快忽慢，盡量斷成短句
4. **減少逗號** — 中文逗號停頓很短，能用句號就用句號
5. **數字/單位口語化** — 例如 `$0.08/hr` → 「一小時零點零八美元」、`6~8%` → 「百分之六到八」
6. **英文專有名詞保留** — 例如 Anthropic、Claude、RAG、API 直接用原文（TTS 會用英文發音念）
7. **加自然語氣** — 適時加入「其實」「說起來」「值得注意的是」讓敘述更流暢自然
8. **開頭結尾要有招呼** — 開頭引入主題、結尾一句收束，不要突然開始也不要突然結束

## 結構建議

- **開頭**：一句引入主題，例如「今天要說的是...」或「接下來這段，要說的是...」
- **中段**：依邏輯順序講述各個重點，一個重點一段，不要把所有事情塞在一起
- **結尾**：一句收束，把這段說完收攏，例如「這就是...的故事」或「大概就是這樣了」

## 內容濃縮策略

- 條列式內容 → 改寫成流暢的敘述句，保留邏輯順序
- 多個重點 → 一次說一個，段落之間用自然語氣銜接
- 數據細節 → 留下關鍵數字，捨棄次要修飾
- 整體長度建議：10~25 段落，對應 5~8 分鐘有聲書

## 操作步驟

1. 讀取使用者提供的原文（可能是純文字、Markdown、書段節錄）
2. 抽出主要主題脈絡，分成 3~5 個邏輯段落
3. 用上述格式撰寫獨白腳本
4. 預設輸出到當前目錄的 `script_<日期>.txt`，檔名可由使用者指定
5. 寫完後告訴使用者下一步可以執行 `audio` skill 生成 mp3
