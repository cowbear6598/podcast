# 工作流程

把使用者提供的文章改寫成雙人對話腳本，輸出可直接餵給 `audio` skill 生成 mp3 的 `.txt` 檔。

## 輸出格式

固定使用 `[曉臻]` 和 `[雲哲]` 標記說話者，一段一行，段落間空行：

```
[曉臻] 大家好，歡迎收聽今天的技術 Podcast，我是曉臻。

[雲哲] 我是雲哲，今天我們要來聊...

[曉臻] 雲哲，可以先幫大家解釋一下嗎？

[雲哲] 沒問題。...
```

## 角色設定

- **曉臻**（女主持人）：負責開場、提問、串場、收尾、表達好奇與反應
- **雲哲**（男主持人）：負責解釋技術細節、補充背景、給出觀點

## 改寫規則

1. **去掉所有 emoji 和表情符號** — TTS 會把 `:fire:` 念成「fire」非常突兀
2. **去掉 URL、`@mention`、Hashtag** — Podcast 用不到，念出來干擾
3. **長句拆短** — Edge TTS 中文長句容易語速忽快忽慢，盡量斷成短句
4. **減少逗號** — 中文逗號停頓很短，能用句號就用句號
5. **數字/單位口語化** — 例如 `$0.08/hr` → 「一小時零點零八美元」、`6~8%` → 「百分之六到八」
6. **英文專有名詞保留** — 例如 Anthropic、Claude、RAG、API 直接用原文（TTS 會用英文發音念）
7. **加自然反應語氣** — 例如「哇」「真的假的」「這也太久」讓對話更像 Podcast 而非互相朗讀
8. **開頭結尾要有招呼** — 開場「大家好歡迎收聽...」、結尾「謝謝大家收聽下次見」

## 內容濃縮策略

- 條列式內容 → 改寫成「曉臻問、雲哲答」的問答對話
- 多個重點 → 一次只談一個，切段處理
- 數據細節 → 留下關鍵數字，捨棄次要修飾
- 整篇長度建議：對話腳本約 25~40 段（換手次數），對應 5~8 分鐘 Podcast

## 操作步驟

1. 讀取使用者提供的原文（可能是純文字、Markdown、URL 內容）
2. 抽出主要事件/重點，分成 3~5 個主題段落
3. 用上述格式撰寫對話腳本
4. 預設輸出到當前目錄的 `script_<日期>.txt`，檔名可由使用者指定
5. 寫完後告訴使用者下一步可以執行 `audio` skill 生成 mp3
