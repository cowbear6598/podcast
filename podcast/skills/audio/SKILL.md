# 工作流程

呼叫 `scripts/podcast.py`，讀取對話腳本，用 Edge TTS 合成兩個聲音（曉臻/雲哲），ffmpeg 裁掉頭尾靜音後合併成單一 mp3。

```bash
python3 <skill-dir>/scripts/podcast.py "對話腳本.txt" -o "輸出.mp3"
```

## 預設設定（寫死在 podcast.py 中）

| 參數 | 預設值 | 說明 |
|------|--------|------|
| 曉臻聲音 | `zh-TW-HsiaoChenNeural` | 台灣女聲 |
| 雲哲聲音 | `zh-TW-YunJheNeural` | 台灣男聲 |
| 語速 | `-10%` | 鎖定避免忽快忽慢，注意只接受兩位數百分比 |
| 段落間停頓 | `0.25` 秒 | 裁掉 TTS 自帶頭尾靜音後再補的停頓 |

## 腳本格式要求

對話腳本必須使用 `[曉臻]` 和 `[雲哲]` 標記說話者：

```
[曉臻] 大家好...

[雲哲] 我是雲哲...
```

如使用者提供的是文章而非對話腳本，應先呼叫 `content` skill 改寫成對話格式。

## 操作步驟

1. 確認腳本檔案存在且格式正確
2. 執行 `podcast.py`
3. 顯示生成進度（會列出每段台詞的前 30 字）
4. 完成後告知使用者輸出檔位置與大小
5. 如果使用者有設定 Slack（`soap-chat:slack`），可詢問是否要直接發送

## 常見問題

- **語速還是忽快忽慢** → Edge TTS 限制，已用 `-10%` 鎖定，極端情況改 `RATE = "-15%"`
- **停頓太久/太短** → 改 `PAUSE_SECONDS`，建議範圍 `0.1` ~ `0.4`
- **個位數百分比報錯** → Edge TTS bug，`RATE` 不可用 `-5%`，要 `-10%` 或 `+0%`
- **無音訊回傳** → 通常是網路問題或聲音名稱拼錯
